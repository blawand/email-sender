How to Use SendSuite & Limits
Choose Your Mode:
  • Send using CSV Data: Best for sending personalized emails based on data in a spreadsheet (CSV file).
  • Manual Recipient List: Best for sending the exact same email to a list of people you type or paste in. No personalization.
CSV Mode Steps:
  1. Upload CSV: Select your `.csv` file. The first row MUST contain headers (e.g., `Email`, `FirstName`). Download Sample CSV.
  2. Check Headers: If valid, detected headers (placeholders) will appear below the upload.
  3. Recipient Template: Enter the placeholder for the email column (e.g., `{Email}`). ONLY ONE placeholder.
  4. Compose Email: Write Subject/Body. Use or buttons/icons to insert placeholders like `{FirstName}`. They must match CSV headers exactly (case-sensitive). Attach files if needed.
  5. Preview: Check how the email looks using data from the *first* CSV row. Verify placeholders.
  6. Send: Sends emails from your logged-in Gmail account.
Manual Mode Steps:
  1. Enter Recipients: Type/paste emails (comma, space, or line separated).
  2. Compose Email: Write Subject/Body & attach files. Note: Text like `{Placeholder}` is sent literally, NOT replaced.
  3. Preview: Check the static email content.
  4. Send: Sends emails from your logged-in Gmail account.
Usage Limits:
  • Rate Limit: Sending is rate-limited. Wait if you hit the limit.
  • CSV Rows: Max {{ limits.MAX_CSV_RECIPIENTS }} rows processed.
  • Manual Recipients: Max {{ limits.MAX_MANUAL_RECIPIENTS }} recipients.
  • Attachments: Max {{ limits.MAX_ATTACHMENTS_PER_EMAIL }} files, Total {{ limits.MAX_TOTAL_ATTACHMENT_SIZE_MB }} MB.
  • Gmail Limits: Google's own sending limits also apply.
General Notes:
  • Authentication: Uses secure Google OAuth2. Your password is never seen/stored.
  • Placeholders (CSV): Must be `{HeaderName}` and match exactly (case-sensitive).
  • Preview: Always review carefully before sending.
  • Errors: Check status messages for details on skipped/failed items.
